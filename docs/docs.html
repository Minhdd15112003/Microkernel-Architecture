Pluggable Backend Architecture

Monorepo — TypeScript — Express — sql.js — In-Memory Event Bus

1. Tổng quan kiến trúc

Dự án theo mô hình Microkernel: một Core Server nhỏ gọn làm trung tâm, các module chức năng cắm vào/rút ra mà không cần restart.

exame/
├── shared-common/        ← Interface contracts (IPlugin, CoreContext, IEventBus)
├── core-server/          ← Microkernel engine (ModuleManager, Dynamic Router, Circuit Breaker)
└── modules/
    ├── auth-module/      ← Module bắt buộc: SQLite users.db, login/register
    └── payment-module/   ← Module tùy chọn: SQLite payment.db, checkout

2. Luồng hoạt động

2.1. Khởi động

  1. runner.ts tạo PluggableServer với đường dẫn ../modules
  2. Constructor tạo: Express app, ModuleManager, CoreContext (console logger + EventEmitter)
  3. start() gọi ModuleManager.scanModules() — quét mỗi thư mục con trong modules/, kiểm tra dist/index.js
  4. Với mỗi module tìm thấy: loadAndMount()

2.2. Load module

ModuleManager.loadModule(moduleDir, context)
 ├─ dynamic import(`file://...dist/index.js?t={timestamp}`)
 ├─ kiểm tra duck-type: module?.name, .initialize, .execute, .shutdown
 ├─ module.initialize(context)   ← mỗi module tự tạo SQLite DB
 ├─ lưu vào Map
 └─ trả về module

PluggableServer.loadAndMount(moduleDir)
 ├─ ModuleManager.loadModule()
 ├─ tạo Router mới
 ├─ module.execute(router)       ← module đăng ký route handler
 ├─ wrapRouterForCircuitBreaker() ← bọc mỗi handler bắt async error
 ├─ app.use('/api/v1/modules/{name}', router)
 └─ lưu router vào moduleRouters map

2.3. Unload module

PluggableServer.unloadAndUnmount(moduleName)
  ├─ xoá router khỏi moduleRouters map
  ├─ filter Express app._router.stack, bỏ layer có path /api/v1/modules/{name}
  │  → request sau đó vào module này nhận 404, không crash
  └─ ModuleManager.unloadModule(name)
     ├─ module.shutdown()   ← module đóng SQLite DB, unsubscribe event bus
     └─ xoá khỏi Map

3. Hot Plugging — Tháo/lắp module nóng

Hot plug = thêm/xoá thư mục module khi server đang chạy, không restart.

3.1. Watcher (chokidar)

ModuleManager.startWatcher() theo dõi pattern modules/*/dist:

Sự kiệnHành động
Thêm thư mục dist/Gọi onLoad(moduleDir) → load + mount
Thêm file index.jsNếu module đã tồn tại: unload + load lại (reload code mới)
Xoá thư mục dist/Gọi onUnload(moduleName) → unmount + shutdown

Mỗi handler bọc trong try-catch riêng → một module lỗi không ảnh hưởng module khác.

3.2. Tại sao không crash?

  • Module load lỗi: loadModule() throw → loadAndMount() trong start() có try-catch per module → log lỗi, skip, server tiếp tục
  • Route handler runtime lỗi: wrapRouterForCircuitBreaker() bọc mỗi handler → async rejection gọi next(err) → error middleware trả về 503 thay vì crash
  • Module unload: xoá layer khỏi Express stack → request sau vào module đó nhận 404 tự nhiên, không undefined route
  • Event emit khi module tắt: EventEmitter.emit() không listener là no-op; module dùng bus?.emit() optional chaining

4. In-Memory Event Bus

4.1. Định nghĩa

Event Bus là cơ chế giao tiếp giữa các module mà không cần import trực tiếp. Một module phát (emit) sự kiện, module khác lắng nghe (subscribe). Hai module hoàn toàn độc lập — không biết đến nhau.

4.2. Cài đặt

  • Interface IEventBus trong shared-common/src/index.ts: on(), off(), emit()
  • Triển khai: Node.js EventEmitter — built-in, không cần thư viện ngoài
  • Instance duy nhất được tạo trong PluggableServer constructor, đưa vào CoreContext
  • Mọi module nhận CoreContext qua initialize(context) → cùng một EventEmitter

4.3. Luồng sự kiện

auth-module                          payment-module
     │                                     │
     │  POST /api/v1/modules/auth/         │
     │  register(username, password)       │
     │       │                              │
     │  INSERT INTO users ...              │
     │  bus.emit('user:created', {         │
     │    username                         │
     │  }) ──────────────────────────────────→
     │                                     │
     │                          onUserCreated({ username })
     │                          INSERT INTO wallets (username, balance)
     │                          → Wallet tạo thành công
     │                                     │
     │  res.json({ success: true })        │

4.4. An toàn khi module tắt

  • Payment-module lưu reference đến handler (this.onUserCreated)
  • Khi shutdown(): gọi bus.off('user:created', handler) — unsubscribe, không leak memory
  • Nếu payment-module chưa load: bus.emit() không có listener → không có gì xảy ra, auth vẫn hoạt động
  • Nếu payment-module bị unload đột ngột: handler đã được gỡ, emit không gây lỗi

5. Đọc code theo thứ tự nào

Mẹo: Đọc theo thứ tự này để hiểu từ contract → engine → module.

  1. shared-common/src/index.ts (28 dòng)

    Bắt đầu ở đây. Đây là hợp đồng giữa Core và Module. Hiểu IPlugin, CoreContext, IEventBus.

  2. core-server/src/module-manager.ts (87 dòng)

    Trái tim của Microkernel: scanModules() quét thư mục, loadModule() dùng dynamic import() có cache-buster, startWatcher() dùng chokidar.

  3. core-server/src/index.ts (90 dòng)

    Express server + Dynamic Router + Circuit Breaker. Xem loadAndMount() và wrapRouterForCircuitBreaker().

  4. modules/auth-module/src/index.ts

    Module mẫu bắt buộc. Xem cách nó triển khai IPlugin, tự tạo SQLite DB trong initialize(), emit sự kiện sau register.

  5. modules/payment-module/src/index.ts

    Module mẫu tùy chọn. Xem cách nó subscribe event bus trong initialize(), unsubscribe trong shutdown().
