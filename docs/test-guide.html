Hướng dẫn kiểm thử Plug-and-Play

Kiểm tra tính năng tháo/lắp module nóng mà không restart server.

Yêu cầu: Node.js 20+, terminal, Postman / REST client bất kỳ

Bước 0: Build toàn bộ dự án

Mở terminal tại thư mục gốc exame/ 
npm install
npm run build

Build thành công = 4 workspace đều in ra tsc không lỗi.

Bước 1: Khởi động Core Server

npm run dev --workspace=@pluggable/core-server

Kết quả mong đợi:

[auth-module] DB ready
Module loaded: auth-module v1.0.0
[payment-module] DB ready, listening for user:created
Module loaded: payment-module v1.0.0
Watching: ...modules\*\dist
Pluggable Backend Server started on port 3000
Loaded: auth-module, payment-module

✓ Cả 2 module được load thành công

Bước 2: Kiểm tra module đang hoạt động

Request:

GET http://localhost:3000/health
GET http://localhost:3000/modules

→ 200
{ "status": "running", "modules": ["auth-module", "payment-module"] }

→ 200
{ "modules": ["auth-module", "payment-module"] }

✓ Core Server sống, 2 modules sẵn sàng

Bước 3: Test API của 2 module

3.1. Register & Login (auth-module)

POST /api/v1/modules/auth-module/register
Content-Type: application/json

{ "username": "alice", "password": "secret" }

→ 200
{ "success": true, "message": "User registered" }


POST /api/v1/modules/auth-module/login
Content-Type: application/json

{ "username": "alice", "password": "secret" }

→ 200
{ "success": true, "token": "mock-jwt-alice" }

✓ Auth-module: register + login, dữ liệu ghi vào auth.db

ℹ Kiểm tra terminal server: in [payment-module] Wallet created for alice — Event Bus hoạt động

3.2. Checkout (payment-module)

POST /api/v1/modules/payment-module/checkout
Content-Type: application/json

{ "amount": 29.99 }

→ 200
{ "success": true, "transactionId": "txn-1", "amount": 29.99, "currency": "USD" }

✓ Payment-module: checkout thành công

Bước 4: THÁO MODULE NÓNG (Hot Unplug)

Mục tiêu: Xoá payment-module khi server đang chạy, kiểm tra Core không crash, auth vẫn hoạt động.

Mở terminal thứ hai (giữ nguyên server đang chạy). Xoá thư mục dist của payment-module: 
rmdir /s /q modules\payment-module\dist

Kết quả trên terminal server:

Module unloaded: payment-module
✓ Module unloaded: payment-module

✓ Server không crash, không restart

4.1. Kiểm tra payment-module đã biến mất

GET /modules

→ 200
{ "modules": ["auth-module"] }

✓ payment-module không còn trong danh sách

4.2. Gọi API payment — phải trả về 404

POST /api/v1/modules/payment-module/checkout
Content-Type: application/json

{ "amount": 10 }

→ 404 Not Found
<!DOCTYPE html>...Cannot POST /api/v1/modules/payment-module/checkout

✓ 404 — route đã bị gỡ, không crash, không undefined

4.3. Kiểm tra auth-module vẫn sống

GET /health

→ 200
{ "status": "running", "modules": ["auth-module"] }


POST /api/v1/modules/auth-module/login
Content-Type: application/json

{ "username": "alice", "password": "secret" }

→ 200
{ "success": true, "token": "mock-jwt-alice" }

✓ Core Server vẫn chạy, auth-module vẫn phục vụ request

4.4. Kiểm tra Event Bus an toàn

Đăng ký user mới — không có payment-module, không crash:

POST /api/v1/modules/auth-module/register
Content-Type: application/json

{ "username": "bob", "password": "pass" }

→ 200
{ "success": true, "message": "User registered" }

✓ Auth vẫn register được, emit vào bus không listener là no-op

Bước 5: LẮP MODULE NÓNG (Hot Plug)

Mục tiêu: Đưa payment-module trở lại, server tự động load mà không restart.

Build lại payment-module: 
cd exame
npm run build --workspace=@pluggable/payment-module

Terminal server tự động in ra:

Module loaded: payment-module v1.0.0
[payment-module] DB ready, listening for user:created

✓ Watcher phát hiện thư mục dist/ mới → tự động load

5.1. Kiểm tra payment hoạt động trở lại

GET /modules

→ 200
{ "modules": ["auth-module", "payment-module"] }


POST /api/v1/modules/payment-module/checkout
Content-Type: application/json

{ "amount": 49.99 }

→ 200
{ "success": true, "transactionId": "txn-1", "amount": 49.99, "currency": "USD" }

✓ Payment-module hoạt động trở lại, không restart server

5.2. Kiểm tra database cũ vẫn còn

Vì payment.db được lưu ở modules/payment-module/payment.db (không nằm trong dist/), dữ liệu cũ không bị mất khi xoá dist/:

node -e "
import('sql.js').then(m=>m.default()).then(SQL=>{
  const {readFileSync}=require('fs');
  const db=new SQL.Database(readFileSync('modules/payment-module/payment.db'));
  console.log(JSON.stringify(db.exec('SELECT * FROM transactions')));
})"

✓ Transactions cũ vẫn còn trong database

Bước 6: Kiểm tra Circuit Breaker

Mục tiêu: Mô phỏng route handler ném lỗi — server không crash, trả về 503.

Tạm thời sửa auth-module để route login throw lỗi: 
// modules/auth-module/src/index.ts — dòng ~30, thêm vào đầu execute():
router.post('/crash', () => { throw new Error('Simulated crash'); });
npm run build --workspace=@pluggable/auth-module

ℹ Watcher phát hiện file index.js thay đổi → tự động reload module

POST /api/v1/modules/auth-module/crash

→ 503
{ "error": "auth-module temporarily unavailable" }

Kiểm tra server vẫn sống:

GET /health

→ 200
{ "status": "running", "modules": ["auth-module", "payment-module"] }

✓ Circuit Breaker hoạt động: error → 503, không crash

Sau khi test, revert lại code, build lại.

# Tắt server: Ctrl+C

# Xoá database test
del modules\auth-module\auth.db
del modules\payment-module\payment.db
